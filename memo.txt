   Creer un compte Render

  - Va sur https://render.com et connecte-toi avec ton compte GitHub




  3. Deployer via Blueprint (methode la plus simple)

  1. Va sur Dashboard → New → Blueprint
  2. Connecte ton repo GitHub FriterieV3
  3. Selectionne la branche br_claude (ou main si tu merges d'abord)
  4. Render detecte automatiquement le render.yaml et propose de creer :
    - friterieshop-db (PostgreSQL gratuit)
    - friterieshop-api (Web Service Docker)
    - friterieshop-web (Web Service Docker)
  5. Clique Apply et attends le deploiement (~5-10 min pour le premier build)

  4. Mettre a jour les URLs reelles

  Une fois les services crees, Render attribue des URLs (ex: friterieshop-api-xxxx.onrender.com). Si elles different du
  nom prevu :

  1. Dashboard → friterieshop-api → copie l'URL reelle
  2. Dashboard → friterieshop-api → Environment → modifie :
    - Jwt__Issuer = https://<url-reelle-api>
    - Jwt__Audience = https://<url-reelle-api>
  3. Dashboard → friterieshop-web → Environment → modifie :
    - API_BASE_URL = https://<url-reelle-api>/api/
  4. Aussi mettre a jour appsettings.Production.json dans le code avec la bonne URL, puis repusher

  5. Verifier

  - Ouvre l'URL du frontend (ex: https://friterieshop-web-xxxx.onrender.com)
  - Teste : navigation, login, ajout au panier

  Rappels tier gratuit

  - Premier chargement lent (~30s) : les services gratuits s'eteignent apres 15 min d'inactivite
  - PostgreSQL : supprime apres 90 jours
  - 512 MB RAM par service
  - Les uploads d'images ne persistent pas entre redeploiements (pas de volume persistant sur le tier gratuit)

  Étapes côté Neon

  1. Va sur https://neon.tech et crée un compte (GitHub login possible)
  2. Clique New Project
  3. Choisis :
    - Project name : friterieshop
    - Region : AWS EU West (Ireland) (le plus proche)
    - PostgreSQL version : 17 (par défaut)
  4. Clique Create Project
  5. Tu verras une connection string du type :
  postgresql://neondb_owner:xxxx@ep-xxx-xxx-123.eu-west-1.aws.neon.tech/neondb?sslmode=require

  Copie-moi cette connection string (ou juste le host, user, password, database) et je configure le tout dans Render
  pour que ton API pointe vers Neon au lieu de la base Render.

  Notre code gère déjà le format URI PostgreSQL grâce au NormalizeConnectionString qu'on a ajouté, donc ça fonctionnera
  directement.
